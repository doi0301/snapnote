## 한 줄 소개 (짧은 버전)

**Windows 우선 데스크톱 메모 앱.** 접어 두는 **폴디드 패널**, 구조화된 줄 단위 편집, 클립보드·히스토리·트레이까지 한 화면 흐름으로 묶었습니다.

---

## 핵심 기능

| 영역 | 설명 |
|------|------|
| **폴디드 패널** | 화면 가장자리에 붙는 얇은 패널. 최대 3개 메모를 스택으로 두고, 호버로 미리보기·클릭으로 바로 편집. |
| **구조화 메모** | 들여쓰기(최대 4단), 체크박스 줄, 굵게/취소선/형광펜 등 **한 메모 안에서** 정리. |
| **히스토리** | 전체 메모 검색·태그 필터(OR)·선택 삭제. 목록은 **본문이 있는 메모** 중심으로 노출. |
| **클립보드 히스토리** | 복사한 텍스트·이미지 흐름을 쌓아 두고, 편집 창에서 **한 번에 삽입**. |
| **시스템 트레이** | 닫아도 백그라운드 상주. 전역 단축키로 패널 표시/숨김, 새 메모·히스토리 바로가기. |
| **설정** | 시작 프로그램, 클립보드 모니터링, 슬롯 색·창 크기, 단축키, 데이터보내기/가져오기. |

## 차별화 가치 (왜 SnapNote인가)

1. **“웹 메모장”이 아니라 데스크톱 워크플로**  
   브라우저 탭과 경쟁하지 않고, **항상 위·에지 스냅·트레이**로 **작업 중인 화면 옆**에 붙어 있습니다.

2. **폴디드 ↔ 편집 ↔ 미리보기가 한 제품 안에서 연결**  
   메모를 “목록에서만” 보지 않고, **접어 둔 상태에서도** 다음 작업(미리보기·편집)으로 자연스럽게 이어집니다.

3. **로컬 우선**  
   메모와 클립보드 히스토리는 **내 PC의 SQLite**에 저장됩니다(구현: `sql.js`). 계정 강제 없이 오프라인에 가깝게 쓸 수 있습니다.

4. **기획·구현이 문서(SRD/TRD/TASKS)와 같이 간다**  
   요구사항이 코드와 어긋나지 않도록 스프린트 단위 작업으로 쪼개 두었고, E2E·성능 검증 스크립트로 회귀를 막습니다.

## 기술 스택 (한 줄)

**Electron + React + TypeScript + electron-vite**, DB는 **sql.js**(WASM SQLite), Windows 배포는 **NSIS** 설치 파일.

---

## 영문 짧은 버전 (선택)

**SnapNote** — A Windows-first desktop memo app with a **folded edge panel** (up to 3 stacked memos), structured line editing, clipboard history, tray integration, and local SQLite storage (`sql.js`). Built with Electron, React, and TypeScript.
