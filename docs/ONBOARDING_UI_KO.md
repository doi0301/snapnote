# SnapNote 온보딩 UI 기획 (초안)

처음 사용하는 사용자에게 핵심 기능을 안내하는 **온보딩(첫 실행 또는 도움말)** 화면/플로우를 위한 기획 문서입니다. 실제 UI 구현 시 이 구조를 기준으로 카피·레이아웃을 조정합니다.

## 목적

- SnapNote가 **무엇을 해 주는 앱**인지 한눈에 전달한다.
- **Tab 들여쓰기**, **부가 기능(클립보드·이모지·텍스트 서식)**, **히스토리**, **설정**을 순서 있게 소개한다.

## 콘텐츠 구성 (섹션)

### 1. 환영 / 한 줄 요약

- Windows 우선 **빠른 메모** 앱, 폴디드 패널·편집 창·히스토리의 관계를 한 문장으로.

### 2. 구조화된 메모 (Tab)

- **Tab**: 줄 단위로 들여쓰기해 목록·계층을 표현할 수 있음.
- **Shift+Tab**: 들여쓰기 줄임.

### 3. 부가 기능

- **클립보드**: 복사한 텍스트·이미지 히스토리(설정에서 모니터링 on/off).
- **이모지·심볼 팔레트**: 이모지 및 특수문자 삽입.
- **텍스트**: 굵게, 취소선, **하이라이트(여러 색)**, 체크박스 줄, 구분선 등.

### 4. 히스토리

- 저장된 메모 목록, **태그**로 묶어 보기, **검색**으로 빠르게 찾기.
- 메모별 **완료(완료)** 표시로 목록에서 시각적으로 구분.

### 5. 설정

- **창 투명도** 등 표시 옵션.
- **데이터 백업**: 메모 내보내기 / 가져오기(설정 화면의 버튼 기준).

### 6. 추후: 스크린샷 슬롯

- 각 섹션마다 **스크린샷 이미지** + **짧은 설명 문구**를 넣을 자리를 확보한다.
- 예: `[스크린샷: 폴디드 패널]` / 캡션 1~2문장.

## 비고

- 실제 온보딩 **화면 구현**(모달, 툴팁 투어 등)은 별도 스프린트에서 결정한다.
- 앱 내 문구는 제품 카피 가이드에 맞춰 다듬는다.
