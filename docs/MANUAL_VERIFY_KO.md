# SnapNote — 수동 확인 체크리스트 (개발/릴리스 전)

`npm run dev` 또는 `npm run build:win` 설치본 기준으로 아래를 순서 없이 점검하면 됩니다.

## 트레이·종료
- [ ] 트레이 아이콘 **Quit** 시 프로세스가 완전히 종료되는지
- [ ] 폴디드 **헤더 ✕**는 숨김만 하고, 트레이에서 다시 열리는지

## 폴디드·가장자리
- [ ] 폴디드를 화면 가장자리 근처로 드래그한 뒤 **손을 떼면** 스냅되는지
- [ ] 스냅된 뒤에도 **다시 드래그하면** 정상적으로 떨어지는지 (모서리에 영구 고정처럼 보이면 안 됨)

## 편집·저장
- [ ] 편집창에서 글 수정 후 **접기(━)** / **스택 제거(✕)** / 폴디드 줄의 **✕**로 닫아도 폴디드 목록·다시 열었을 때 내용이 유지되는지
- [ ] 타이핑 후 **약 1.5초 뒤** DB에 반영되는지(자동 저장)
- [ ] **트레이 Quit** 시 열린 편집창 내용이 유실되지 않는지
- [ ] (선택) **Alt+F4**로 편집창만 닫을 때는 OS 동작에 따름

## 호버 미리보기
- [ ] 메모 **줄에만** 올렸을 때(약 150ms 후) 미리보기가 **폴디드 패널 바로 옆**에 뜨는지
- [ ] 포인터가 **그 줄을 벗어나면** 미리보기가 곧바로 사라지는지(약 120ms, 프리뷰로 이동 시 유지)
- [ ] 미리보기 **클릭** 시 해당 메모 편집 창이 열리는지

## 빌드
- [ ] `npm run typecheck` / `npm test` 통과 (CI 또는 로컬)
